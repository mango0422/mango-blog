# 1️⃣ 정규화 / 키 요약 + 설명

지하철에서 3~5회 반복하면 충분히 외워짐.

## ■ 정규화 핵심 3줄 (⚡암기)

- **1NF: 원자값 / 반복속성 제거** → 하나의 칸에는 하나의 값만.
- **2NF: 부분 함수 종속 제거** → 기본키 일부에만 종속된 컬럼 금지.
- **3NF: 이행 함수 종속 제거** → 비키가 비키를 결정하면 안 됨.

### ▷ 설명

정규화는 **중복 제거 + 이상현상 해결**을 위한 절차.

시험에서는 "무슨 종속 때문에 몇 정규형 위반인가?"를 묻는 패턴이 압도적.

- 2NF 위반 = 기본키가 복합키 + 일부키 → 종속되는 컬럼 존재
- 3NF 위반 = A→B, B→C 구조에서 C가 A에 종속된 것처럼 보이는 경우
- "결정자 / 함수 종속" 단어가 보이면 정규화 문제라고 생각하면 됨

---

## ■ BCNF 3줄 (⚡암기)

- **모든 결정자가 후보키인 정규형.**
- **3NF보다 더 엄격함.**
- **후보키가 복수일 때 위반이 자주 발생.**

### ▷ 설명

3NF는 "비키가 비키를 결정"하면 위반이지만, BCNF는 "후보키가 아닌 컬럼이 결정자"로 등장해도 위반.

즉, **결정자 = 반드시 후보키**여야 한다.

---

## ■ 키(Key) 3줄 정리 (⚡암기)

- **후보키: 최소성을 만족하며 유일하게 구분 가능.**
- **기본키: 후보키 중 선택된 것.**
- **대체키: 기본키로 선택되지 않은 후보키.**

### ▷ 설명

시험에서 가장 많이 틀리는 영역.

- 후손 개념: 후보키 → 기본키 선택 → 나머지가 대체키
- 외래키는 부모 테이블 기본키를 참조하며 NULL 가능 여부가 문제로 출제됨.

---

# 2️⃣ JOIN / 집합연산 요약 + 설명

JOIN은 그림처럼 머릿속에 구조를 떠올리는 게 핵심.

## ■ JOIN 4종 (⚡암기)

- **INNER JOIN: 조건 일치하는 행만**
- **LEFT JOIN: 왼쪽 전체 + 오른쪽 매칭**
- **RIGHT JOIN: 오른쪽 전체 + 왼쪽 매칭**
- **FULL OUTER JOIN: 양쪽 전체**

### ▷ 설명

LEFT JOIN 문제의 80%는 이렇게 푼다:

> LEFT 결과 = INNER 결과 + LEFT 단독 데이터
> 

시험에서는 다음 패턴으로 나온다:

- A 테이블 기준으로 모두 출력해야 한다 → LEFT JOIN 사용
- 양쪽 없는 값도 포함해야 한다 → FULL OUTER JOIN
- 매칭되는 행만 필요하다 → INNER JOIN

---

## ■ 집합 연산 4종 (⚡암기)

- **UNION: 중복 제거**
- **UNION ALL: 중복 포함 (빠름)**
- **INTERSECT: 교집합**
- **MINUS: 왼쪽 - 오른쪽 차집합**

### ▷ 설명

헷갈리면 다음 원칙으로 해결된다:

- 속도 가장 빠름 = UNION ALL
- 중복 제거 O = UNION / INTERSECT
- 방향성 있음 = MINUS

문제에서 다음 문구가 나오면 바로 MINUS다:

> A에는 있는데 B에는 없는 값
>

# 3️⃣ 함수(Function) 10개 공식 + 예시 + 암기 포인트

함수는 직접 손가락 계산이 가능해야 함.

## ■ 문자열 함수 (⚡암기)

- **SUBSTR(str, s, n)**: s번째 문자부터 n개
    
    예: SUBSTR('ABCDE', 2, 3) → "BCD"
    
- **INSTR(str, substr, start)**: 부분문자 위치
    
    예: INSTR('ABCDAB', 'AB', 2) → 5
    

### ▷ 설명

시험은 "몇 번째인지"가 아니라 **정확한 숫자**를 물어본다.

---

## ■ 숫자 함수 (⚡암기)

- **ROUND(n, d)**: d자리에서 반올림
    
    예: ROUND(12.345, 2) → 12.35
    
- **TRUNC(n, d)**: d자리에서 버림
    
    예: TRUNC(12.345, 2) → 12.34
    

### ▷ 설명

ROUND vs TRUNC 구분만으로 2문제는 확보됨.

---

## ■ 날짜 함수 (⚡암기)

- **ADD_MONTHS(date, n)**: n개월 더하기
- **MONTHS_BETWEEN(d1, d2)**: 개월 차이 (d1 - d2)

### ▷ 설명

시험 책임 패턴:

- "전년 동월" → ADD_MONTHS(col, -12)
- 소수점 있는 MONTHS_BETWEEN 값 그대로 둬야 하는 문제 출제됨

---

## ■ 변환 함수 (⚡암기)

- **TO_CHAR(값, '포맷')**
- **TO_DATE(문자, '포맷')**
- **CAST(값 AS 타입)**

### ▷ 설명

데이터 타입 변환 문제가 자주 출제됨:

- 숫자를 문자로 바꿈 = TO_CHAR
- 문자열을 날짜로 바꿈 = TO_DATE

---

## ■ NULL 처리 함수 (⚡암기)

- **NVL(a, b)**: a가 NULL이면 b
- **NVL2(a, b, c)**: a가 NULL 아니면 b / NULL이면 c
- **COALESCE(a, b, c...)**: 처음 NULL 아닌 값

### ▷ 설명

NVL / NVL2 전환 문제는 암기 룰로 해결:

```
NVL2 = 조건이 2개인 NVL
(a가 NULL 아니면 b, NULL이면 c)

```

# ✨ 암기용 마무리 요약 (지하철에서 5분 반복용)

### ■ 정규화

- **1NF 원자값 / 2NF 부분종속 제거 / 3NF 이행종속 제거**
- **BCNF: 모든 결정자 후보키**

### ■ 키

- **후보키 → 기본키 선택 → 나머지 대체키**

### ■ JOIN

- **INNER: 매칭만**
- **LEFT: 왼쪽 전체**
- **RIGHT: 오른쪽 전체**
- **FULL: 양쪽 전체**

- **UNION = 중복 제거**
- **UNION ALL = 중복 포함**
- **INTERSECT = 교집합**
- **MINUS = 왼쪽 - 오른쪽**

### ■ 함수 10개

- **SUBSTR / INSTR**
- **ROUND / TRUNC**
- **ADD_MONTHS / MONTHS_BETWEEN**
- **TO_CHAR / TO_DATE / CAST**
